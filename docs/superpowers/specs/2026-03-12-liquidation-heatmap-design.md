# 청산 히트맵 대시보드 설계서

## 개요

바이낸스 선물(BTCUSDT) 실시간 청산 데이터를 수집하여 히트맵으로 시각화하는 대시보드.
Node.js 백엔드에서 WebSocket으로 데이터를 수집·집계하고, 프론트엔드에서 Canvas로 렌더링한다.

## 아키텍처

```
바이낸스 WebSocket API
        │
        ▼
┌───────────────────┐
│  Node.js Server   │
│  (Express + WS)   │
│                   │
│  - 청산 이벤트 수신  │
│  - 가격대별 집계     │
│  - 타임프레임별 버킷  │
│  - 메모리 저장       │
└────────┬──────────┘
         │ WebSocket
         ▼
┌───────────────────┐
│  Frontend         │
│  (HTML + Canvas)  │
│                   │
│  - 히트맵 렌더링    │
│  - 캔들차트 오버레이 │
│  - 롱/숏 분리 뷰    │
│  - 타임프레임 전환   │
└───────────────────┘
```

## 백엔드 설계

### 디렉토리 구조

```
server/
├── index.js              # Express + WebSocket 서버 진입점
├── services/
│   ├── binance-ws.js     # 바이낸스 WebSocket 연결 및 청산 이벤트 수신
│   ├── aggregator.js     # 가격대별·타임프레임별 청산 데이터 집계
│   └── candle-feed.js    # 바이낸스 REST API로 캔들 데이터 조회
└── package.json
```

### 바이낸스 데이터 수신 (binance-ws.js)

- `wss://fstream.binance.com/ws/btcusdt@forceOrder` 엔드포인트 사용
- 청산 이벤트에서 추출하는 필드:
  - `side`: BUY(숏 청산) / SELL(롱 청산)
  - `price`: 청산 가격
  - `quantity`: 청산 수량
  - `timestamp`: 시간
- 자동 재연결 로직 포함

### 데이터 집계 (aggregator.js)

- **가격 버킷**: 현재가 기준 상하 일정 범위를 가격대(예: $50 단위)로 나눔
- **시간 버킷**: 타임프레임별(5m, 15m, 1h, 4h, 1d) 시간 구간으로 나눔
- **저장 구조**:
  ```js
  {
    timeframe: '1h',
    buckets: [
      {
        time: 1710000000000,     // 시간 구간 시작
        priceLevels: {
          '67000': { long: 12.5, short: 3.2 },  // BTC 수량
          '67050': { long: 0.8, short: 15.3 },
          ...
        }
      },
      ...
    ]
  }
  ```
- **메모리 관리**: 타임프레임별 최대 보관 개수 설정 (5m: 288개=24시간, 1d: 30개=1개월)

### 캔들 데이터 (candle-feed.js)

- 바이낸스 REST API `/fapi/v1/klines` 사용
- 히트맵 타임프레임에 맞춰 캔들 데이터 조회
- 프론트엔드 요청 시 또는 주기적으로 갱신

### WebSocket 서버 → 프론트엔드

- Express 서버에 `ws` 라이브러리로 WebSocket 서버 구성
- 클라이언트 연결 시:
  1. 현재 집계된 히트맵 스냅샷 전송
  2. 캔들 데이터 전송
  3. 이후 실시간 청산 이벤트를 스트리밍
- 메시지 타입:
  - `snapshot`: 초기 전체 데이터
  - `liquidation`: 개별 청산 이벤트
  - `candles`: 캔들 데이터

### 뷰 모드

1. **히트맵 단독**: 히트맵만 표시
2. **캔들차트 + 히트맵 오버레이**: 캔들차트 위에 히트맵을 반투명 오버레이
3. **토글로 전환**: 상단 버튼으로 모드 전환

### 롱/숏 뷰

- **전체**: 롱+숏 합산, 단일 색상 그라디언트 (파랑→노랑→빨강)
- **롱 청산**: 롱 청산만 표시, 빨강 계열
- **숏 청산**: 숏 청산만 표시, 초록 계열

### 히트맵 렌더링

- Canvas 2D API 사용
- 색상: 물량이 적을수록 어둡고, 많을수록 밝고 따뜻한 색
  - 전체 뷰: `#0a0e1c` → `#1a3a5c` → `#f0c040` → `#ff3d3d`
  - 롱 뷰: `#1a0a0a` → `#ff3d3d`
  - 숏 뷰: `#0a1a0a` → `#00ff87`
- X축: 시간 (타임프레임에 맞춤)
- Y축: 가격대
- 각 셀: 해당 시간·가격 구간의 청산 물량에 비례한 색상

### 캔들차트 렌더링

- 기존 index.html의 캔들 렌더링 로직 재활용
- 히트맵과 동일한 X축(시간), Y축(가격) 스케일 공유
- 오버레이 모드에서는 캔들 위에 히트맵을 `globalAlpha` 조절하여 중첩

## 확장 포인트

이후 풀스택으로 확장 시:

| 현재 (Phase 1) | 확장 (Phase 2+) |
|---------------|----------------|
| 메모리 저장 | PostgreSQL/TimescaleDB 저장 |
| BTCUSDT 단일 | 멀티 심볼 (ETH, SOL 등) |
| HTML + Canvas | Next.js + React 전환 |
| 실시간만 | 히스토리 조회 |
| 뷰 전환만 | 가격 알림, 텔레그램 연동 |

확장을 위해 백엔드 서비스 레이어를 분리하고, 프론트엔드 렌더링 로직을 모듈화한다.

## 기술 스택

| 구분 | 기술 |
|------|------|
| 런타임 | Node.js |
| HTTP 서버 | Express |
| WebSocket (서버) | ws |
| WebSocket (클라이언트) | 브라우저 내장 WebSocket |
| 시각화 | Canvas 2D API |
